1. Overview

With Constructor and Destructor pair, people have capability specify Class in following ways:

  • Control states of Object or states of Context during or after construction/destruction of objects(start/off, enable/disable, etc).
  • Manage resources during life cycle of object – RAII (Resource Acquisition Is Initialization).
  • Estabilish the invariant of object.

2. Implicitly-declared default constructor

Implicitly-defiend default constructor willl calls the default constructors of the bases and of the non-static members of this class.

2.1. Deleted Implicitly-declared default constructor

If there are some something that will cause those operations failed then the implicitly-defined default constructor be deleted, e.g,

  • Has a member of reference type without a default initializer.
  • Base class’ default constructor is deleted, inaccessible, ambiguous or
  • Has non-static data member with deleted, inaccessible, ambiguous default constructor.
  • Has a direct or virtual base or a non-static data member which has a deleted destructor or inaccessable destructor.
  • Has a non-const-default-constructible const member without a default member initializer.

5. Class Object Initialization

5.1. Without Constructor

Builtin type has no constructors but still able to initialize it with a value of suitable type. 

int a{}; // Default initialization
int a{1};
char *p{nullptr};

similarly, initialize an object without explicit constructor defined can be initialize in following ways:

  • Memberwise Initialization
  • Copy Initialization
  • Default Initialization

for example: 

struct X {
    int a;
    int b;
    int c;
};

X x1 {1, 2, 3}; / Memberwise Initialization X x2 {x1}; / Copy Initialization X x3 {}; // Default Initialization

5.2. With Constructor

Initialize object with an explicit constructor can establish the invariant of object.

Initialize can be done by three notations:

  • {}, Universal initializer and can be unsed in anywhere for initialize purposes.
  • (), In some case, must use () to indicate that to initialize a list of elements.
#include <vector>

std::vector<int> v1{77}; // Initialize v1 with an element which value is 77.
std::vector<int> v2(77); // Initialize v2 with 77 elements and all elements with value 0.
  • =

Default constructor is the constructor without any arguments. 

struct X {
    X(int a);
    X(double a);
};

X x({}); // ERROR: Which will incur an error due to
         //        ambiguous of constructor selection.

5.2.1. Initialization Order

The order of member initializers in the list is irrelevant: the actual order of initialization is as follows:

  • If the constructor is for the most-derived class, virtual bases are initialized in the order in which they appear in depth-first left-to-right traversal of the base class declarations (left-to-right refers to the appearance in base-specifier lists)
  • Then, direct bases are initialized in left-to-right order as they appear in this class’s base-specifier list
  • Then, non-static data member are initialized in order of declaration in the class definition.

  • Finally, the body of the constructor is executed

    Note: if initialization order was controlled by the appearance in the member initializer lists of different constructors, then the destructor wouldn’t be able to ensure that the order of destruction is the reverse of the order of construction

5.3. initializerlist Constructor

A constructor that takes only one argument of type std::initializerlist<T> is called initializerlist Constructor.

Object of a class type can be initialized by initializer-list if a Initializerlist Constructor is declared and defined by the Class.

Examples: 

vector<double> v = { 1, 2, 3.456, 99.99 };
list<pair<string,string>> languages = {
{"Nygaard","Simula"}, {"Richards","BCPL"}, {"Ritchie","C"}
};

map<vector<string>,vector<int>> years = {
{ {"Maurice","Vincent", "Wilkes"},{1913, 1945, 1951, 1967, 2000} },
{ {"Mar tin", "Richards"} {1982, 2003, 2007} },
{ {"David", "John", "Wheeler"}, {1927, 1947, 1951, 2004} }
};

This is done by declared an Initializer-List Constructor. 

struct X {
    X(initializer_list<int>);
};

X x1{1};

5.3.1. initializerlist Constructor Disambiguation

struct X {
    X(initializer_list<int>);
    X();
    X(int);
};

X x0 {};   // empty list: default constructor or initializer-list constructor? (the default constructor)
X x1 {1}; // one integer: an int argument or a list of one element? (the initializer-list constructor)

Rules to selectd constructor:

  • If either a default constructor or an initializer-list constructor could be invoked, prefer the default constructor.

  • If both an initializer-list constructor and an ‘‘ordinary constructor’’ could be invoked, prefer the initializer-list constructor.

    Cause to keep semantic of different number of argument of initializerlist Construction is necessary.

5.6. static Member Initialization

In general, a static member declared in class and define in outside of class. 

class Node {
    // ...
    static int node_count; // declaration
};
int Node::node_count = 0; // definition

but there are some special case 

class Curious {
public:
    static const int c1 = 7;  // OK
    static int c2 = 11;  // error: not const
    const int c3 = 13;  // OK, but not static (§17.4.4)
    static const int c4 = sqrt(9); // error: in-class initializer not constant
    static const float c5 = 7.0; // error: in-class not integral (use constexpr rather than const)
    // ...
};
