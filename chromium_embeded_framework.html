1. Concepts

1.1. Processes

  • Browser process: Window creation, UI and network access and majority of the application logic.
  • Render process: Blink rendering and JavaScript execution.
  • GPU process: gpu accelerated.

Browser and render processes can communicate by sending async messages back and forth.

1.2. Threads

Each process in CEF3 runs multiple threads. These are some of the commonly-referenced threads:

  • TIDUI: Main thread in the browser process.
  • TIDOI: Thread in browser process to handle IPC and network messages.
  • TIDFILE*: Threads are used in the browser process to interact with the file system.
  • TIDRENDERER: Main thread in the renderer process.

2. Application Structure

Every CEF3 application has the same general structure.

  • Entry-point function that initializes CEF and runs either sub-process executable logic or the CEF message loop.
  • Implementation of CefApp to handle process-specific callbacks
  • Implementation of CefClient to handle browser-instance-specific callbacks.
  • Call CefBrowserHost::CreateBrowser() to create a borwser instance and manage the browser life span using CefLifeSpanHandler.

2.1. Entry-Point Function

A place where the process begin to run.

As we know, CEF3 application will run multiple processes. The processes can

  • All use the same executable or
  • A seperate executable can be specified for the sub-processes

2.1.1. Single Executable

Support on Windows and Linux but not on MacOS

Example: 

// Program entry-point function.
int main(int argc, char* argv[]) {
  // Structure for passing command-line arguments.
  // The definition of this structure is platform-specific.
  CefMainArgs main_args(argc, argv);

  // Implementation of the CefApp interface.
  CefRefPtr<MyApp> app(new MyApp);

  // Execute the sub-process logic, if any. This will either return immediately for the browser
  // process or block until the sub-process should exit.
  int exit_code = CefExecuteProcess(main_args, app.get());
  if (exit_code >= 0) {
    // The sub-process terminated, exit now.
    return exit_code;
  }

  // Populate this structure to customize CEF behavior.
  CefSettings settings;

  // Initialize CEF in the main process.
  CefInitialize(main_args, settings, app.get());

  // Run the CEF message loop. This will block until CefQuitMessageLoop() is called.
  CefRunMessageLoop();

  // Shut down CEF.
  CefShutdown();

  return 0;
}

2.1.2. Separate Sub-Process Executable

Need two seperate executable projects and two seperate entry-point functions.

Main application entry-point function: 

// Program entry-point function.
int main(int argc, char* argv[]) {
  // Load the CEF framework library at runtime instead of linking directly
  // as required by the macOS sandbox implementation.
  CefScopedLibraryLoader library_loader;
  if (!library_loader.LoadInMain())
    return 1;

  // Structure for passing command-line arguments.
  // The definition of this structure is platform-specific.
  CefMainArgs main_args(argc, argv);

  // Implementation of the CefApp interface.
  CefRefPtr<MyApp> app(new MyApp);

  // Populate this structure to customize CEF behavior.
  CefSettings settings;

  // Specify the path for the sub-process executable.
  CefString(&settings.browser_subprocess_path).FromASCII(“/path/to/subprocess”);

  // Initialize CEF in the main process.
  CefInitialize(main_args, settings, app.get());

  // Run the CEF message loop. This will block until CefQuitMessageLoop() is called.
  CefRunMessageLoop();

  // Shut down CEF.
  CefShutdown();

  return 0;
}

Sub-process application entry-point function: 

// Program entry-point function.
int main(int argc, char* argv[]) {
  // Initialize the macOS sandbox for this helper process.
  CefScopedSandboxContext sandbox_context;
  if (!sandbox_context.Initialize(argc, argv))
    return 1;

  // Load the CEF framework library at runtime instead of linking directly
  // as required by the macOS sandbox implementation.
  CefScopedLibraryLoader library_loader;
  if (!library_loader.LoadInHelper())
    return 1;

  // Structure for passing command-line arguments.
  // The definition of this structure is platform-specific.
  CefMainArgs main_args(argc, argv);

  // Implementation of the CefApp interface.
  CefRefPtr<MyApp> app(new MyApp);

  // Execute the sub-process logic. This will block until the sub-process should exit.
  return CefExecuteProcess(main_args, app.get());
}
