1. States

  • System: A collection of rules.
  • Deterministic System: Initial state determine the final state over times.
  • Non-Deterministic System: Initial state only fixing the class of final state.
  • Postcondition: The correct state after an activation of a System.
  • Weakest Precondition: The weakest condition that able to reach the final state which satisfy Postcondition R under a System S. Denote as \(wp(S,R)\) where \(S\) is system and \(R\) is Postcondition. Note: both \(wp(S,R)\) and \(R\) are boolean expressions, so it valid to write \(wp(S,F) = F\) which means there are no state can reach to the state F.

We can write those in Hoare logic:

\(\{wp(S,R)\}S\{R\}\)

While building a system you should know the Postcondition after system run, which is what you want or required. After that you may try to reasoning the Precondition of the system. This process is done by Predicate Transformer

1.1. Predicate Transformer

\(wp\) is Predicate Transformer which transform Postcondition to Weakest Precondition.

1.1.1. Property

  • Property 1. \(\forall{S}: wp(S,F) = F\) where \(F\) is False.
  • Property 2. \(\forall{S}, \forall{Q}, \forall{R}: (Q \Rightarrow R) \Rightarrow (wp(S,Q) \Rightarrow wp(S,R))\) where Q and R are Postcondition.

  • Property 3. \(\forall{S},\forall{Q},\forall{R}: (wp(S,Q) \wedge wp(S,R)) \iff wp(S, Q \wedge R)\) Further more, \((Q \wedge R) \Rightarrow Q\) and follow the Property 2 we have:

    \(wp(S,Q \wedge R) \Rightarrow wp(S,Q)\) \(wp(S,Q \wedge R) \Rightarrow wp(S,R)\)

  • Property 4. \(\forall{S},\forall{Q},\forall{R}: (wp(S,Q) \vee wp(S,R)) \Rightarrow wp(S, Q \vee R)\) Due to Property 2, apply Property 2 to \(Q \Rightarrow (Q \vee R)\) and you can get this property
  • Property 4’. \(\forall{S},\forall{Q},\forall{R}: (wp(S,Q) \vee wp(S,R)) \iff wp(S, Q \vee R)\) where \(S\) is a deterministic system. Hoare logic of deterministic system: \(\{wp(DS,Q \vee R\}S\{Q\}\) or \(\{wp(DS,Q \vee R\}S\{R\}\) and non-deterministic system: \(\{wp(NS, Q \vee R\}S\{Q \vee R\}\)
