3. Service examples

Service can depend on other services. 

export class HeroService {
    private heroes: Hero[] = [];

    constructor(
    private backend: BackendService,
    private logger: Logger) { }

    getHeroes() {
    this.backend.getAll(Hero).then( (heroes: Hero[]) => {
        this.logger.log(`Fetched ${heroes.length} heroes.`);
        this.heroes.push(...heroes); // fill cache
    });
    return this.heroes;
    }
}

4. Dependency injection (DI)

DI, Angular inject service to component. To enable this action should use the @Injectable() decorator to provide the metadata that allow angular to jnject it into a componenta as a dependency.

  • The injector is the main mechanism. Angular creates an application-wide injector for you during the bootstrap process, and additional injectors as needed. You don’t have to create injectors.
  • An injector creates dependencies, and maintains a container of dependency instances that it reuses if possible.
  • A provider is an object that tells an injector how to obtain or create a dependency.

A Component that depend on HeroService: 

constructor(private service: HeroService) { }

4.1. Step to inject

  • Discovers that a component depends on a service
  • Checks if the injector has any existing instances of that service.
  • Make on if not exists then add to injector before returning the service to Angular.
  • When all requested services have been resolved and returned, angular call component’s constructor with those services as arguments.

5. Providing services

At least one provider of any service that you are going to use is registered. Register providers in the metadata of the service, or in the @NgModule() or @Component() metadata.
