1. Terminology

Proposition: Statements which we can prove, disprove, assume, negate, and so on.

Assumption: Proposition of the for x: A, where x is a variable and A is a type. For example, m + n : N under the assumption m,n: N.

Context: Collection of all such assumptions.

2. Foundations

2.1. Type theory versus Set theoy

2.1.1. Layers

Set theory has two foundation layers:

  • Deductive system of first-order logic

    (Note: Informally, a deductive system is a collection of rules for deriving things called judgments)

  • Axioms of a particular theory

Thus set theory is not only about sets, but rather about the interplay between sets (the objects of the second layer) and propositions (thie objects of the first layer).

By contrast, type theory is its own deductive system, it need not be formulated inside any superstructure, such as first-order logic. That means there is only one layer of type theory.

Type theory has only one layer:

  • Deductive system by its own

2.1.2. Basic notions

another difference is basic notions between set theory and type theory.

Set theory has two basic notions:

  • Set
  • Proposition

and type theory has only basic notions:

  • Proposition (Type)

2.1.3. Essential

A thing which is represent as \(a:A\), the term a has type A. If you treate A more like a set than like a proposition, then \(a:A\) may be regarded as \(a \in A\). However, there is an essential difference in

  • that \(a:A\) is a judgment whereas \(a \in A\) is a proposition.

A good way to think about this is that in set theory, “membership” is a relation which may or may not hold between two pre-existing objects “a” and “A”, while in type theory we cannot talk about an element “a” in isolation: every element by its very nature is an element of some type, and that type is (generally speaking) uniquely determined.

We cannot introduce a variable without specifying it’s type.

2.1.4. Equality

Notion of equality in mathematics is a proposition, can be assumed, negate, prove and disprove.

But notion of equality in type theory there are two kinds of equality

  • (Propositionally) equal
  • Judgmental Equality

(Propositionally) equal is a proposition so it’s a type. For example:

There exists elements \(a:A\) and \(b:A\) then we have a type \(a =_A b\). When \(a =_A b\) is inhabited, we say that a and b are (propositional) equal.

Equality of type theory that showed above is same as equality of set theory. But type theory has another kind of equality which is Judgmental equality which is not a proposition and write it as \(a \equiv b\) or \(a \equiv_{A} b\).

Thus, type theory is a deductive system based on two froms of judgment:

  • \(a:A\) : a is an object of type A
  • \(a \equiv_{A} b\): a and b are definitionally equal objects of type A

3. Function types

Ways to construct elements of \(a \rightarrow B\):

  • Direct definition \(f(x) :\equiv \Phi\) where \(\Phi\) is an expression
  • \(\lambda-abstraction\) \((\lambda(x:A). \Phi): A \rightarrow B\)

4. Universes and families

universe is a type whose elements are types, write as \(\mathcal{U}\).

Function \(B : A \rightarrow \mathcal{U}\) is called families of types (or sometimes dependent types).

5. Dependent function types (\[\Pi\]-types)

The elements of \(\Pi-Types\) are functions whose codomain can be vary depending on elements of domain where the function is applied, called dependent functions.

Given a type \(A : \mathcal{U}\) and a family \(B : A \rightarrow \mathcal{U}\), we may construct the type of dependent functions \(\Pi_{(x : A)} B(x) : \mathcal{U}\).

Explicit definitions:

\(f : \Pi_{(x:A)}B(x)\) or \(f(x) :\equiv \Phi\) for \(x : A\), \(\Phi : B(x)\)

or use lambda-abstraction:

\(\lambda x. \Phi : \Pi_{(x:A)}B(x)\)

5.1. Several Arguments

With curring to define dependent functions with several arguments.

Given \(A:\mathcal{U}\) and type families \(B : A \rightarrow \mathcal{U}\) and \(C : \Pi_{(x:A)}B(x) \rightarrow U\) we have \(\Pi_{(x:A)}\Pi_{(y:B(x))}C(x,y)\)

6. Product type

Given types \(A,B : \mathcal{U}\) we introduce the type \(A \times B : \mathcal{U}\), which is called cartesian product.

There is a nullary product type, \(1: \mathcal{U}\).

6.1. Recursor

Recursor is represented in the form:

\[ rec_{A \times B}: \prod_{C:U} (A \rightarrow B \rightarrow C) \rightarrow A \times B \rightarrow C \] \[rec_{A \times B}(C,g,(a,b)) :\equiv\ g(a)(b)\], where \(g : A \rightarrow B \rightarrow C\)

Thus, we are able to define pr1 and pr2 like following:

\(pr1 : A \rightarrow B \rightarrow A\) \(pr1 :\equiv rec_{A \times B}(A, \lambda a. \lambda b. a, (a,b))\)

\(pr2 : A \rightarrow B \rightarrow B\) \(pr2 :\equiv rec_{A \times B} (B, \lambda a. \lambda b. b, (a,b))\)

Recursor for the unit type:

\[rec_{1} : \prod_{C:\mathcal{U}} C \rightarrow 1 \rightarrow C\]

\[rec_{1}(C,c,*) :\equiv c\]

Dependent functions over the product type:

Given \(C : A \times B \rightarrow \mathcal{U}\) we may define a function \(f : \prod_{x : A \times B} C(x)\) by providing a function:

\[g : \prod_{x : A}\prod_{y : B}C((x,y))\] \[f((x,y)) :\equiv g(x)(y)\]

6.2. Induction

Induction for product types (Why it’s called induction):

\[ind_{A \times B} : \prod_{C : A \times B \rightarrow U} (\prod_{x : A} \prod_{y : B} C(x, y)) \rightarrow \prod_{x : A \times B} C(x) \]

\[ind_{A \times B}(C, g, (a, b) :\equiv g(a)(b)\]
