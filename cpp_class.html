1. Introduction

OOP is a design philosphy. OOP focus on Concepts be involved in their program and relationship among them and also support extra properties that presentation of Concepts, Encauslation, Polymorphism, Inheritence.

This document is talking about how CPP support such design philosphy.

Class provide a way to user to define types in CPP , which provide mechanism to user to apply constrains to their programs for correctness. Class also represent as a concept.

Class has two kinds of members:

  • Data members
  • Member functions
  • Type members

With Data member we are able to provide State to a new type and with Member function we are able to provide actions to a new type. A type has internal state and could be as operand in operations can be create from define a class with both members.

In the following sections we will introduce

  • Structure of a class
  • Basic facilities to definning a class
  • Construction, Cleanup, Copy and Move of class
  • Operator Overloading
  • Derived Classes

3.4. Default Copying

Defaulted Copy Constructor Copy object with Memberwise Copy.

For union types, the implicitly-defined copy constructor copies the object representation (as by std::memmove). For non-union class types (class and struct), the constructor performs full member-wise copy of the object’s bases and non-static members, in their initialization order, using direct initialization.

3.4.1. The Type of Defaulted Copy Constructor:

  • T::T(const T&)

    if able to cosntruct it’s base class and data member with Copy Constructor which arguments with const quantifier or const volatile quantifier.

  • T::T(T&)

    if above rules are not satisfied.

3.4.2. When default copy constructor deleted

if T has

  • non-static data members
  • direct or virtual base class

that cannot be copied (have deleted, inaccessible, or ambiguous copy constructor).

or

if T has direct or base class or a non-static data member with a delted or inaccessible destructor.

or

T has a data member of rvalue reference type.

or

T has a user-defined move constructor or move assignment operator.

3.4.3. Category of Defaulted COpy Constructor:

  1. Trivial copy constructor

    Do nothing but copy raw bytes.

  2. Eligible copy constructor

    A copy constructor is eligible if

    • not deleted
    • constraints satisfied
    • most constrained one in all Copy Constructors.

4.2. Copy and Move

Mechanisms to support semantic ’Value Transfer (x=y)’ are:

  • Copy: To transfer value from one to another by make a duplicate of given value
  • Move: To transfer value from one to another by transfer the ownership of physical part of value.

’Value Transfer’ is the commonality of those two mechanisms. There are differences between them, Copy do not affect the entity that copy from but Move do, Move will let the copy from entity in to Move-From Stated, this state only guarantee correctly of copy (copy-from, copy-to) and destruction.

There are another subtle properties that a copy operation should obey:

  • Equivalence: x’s value must be equal to y’ after ’x=y’.
  • Indenpendence: Change on each of x and y should not change another after ’x=y’.

see also Copy and Move.
