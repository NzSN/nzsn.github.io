2.2. Structural Patterns

Patterns of this category is all about to reach goal by modify structure of Class or a collection of classes and SOLID Principles remain hold.

Our goals may some of following:

  • Ehance types (Decorator, Proxy)
  • State Mixing (Facade)
  • Presentation (Adapter)
  • Interact (Bridge)
  • Deal with Collection of values (Composite)

2.2.1. Adapter

Change the structure of informations or datas from outside view of classes. In short, Adapter is a structure that help transform type signature of one interface into another type. 

class Point { ... };
class Line {
    vector<Point> getPoints() const;
private:
    vector<Point> points;
};
class Rectangle {
public:
    vector<Line> getLines() const;
private:
    vector<Line> lines;
};

// Interface to draw a shape on screen.
void drawing(vector<Point>::iterator start, vector<Point>::iterator end) { ... }

class LineToPointAdapter {
public:
    using Points = vector<Point>;

    LineToPointAdapter(Line &line) {
        points = line.getPoints();
    }

    virtual Points::iterator begin() {
        return points.begin();
    }

    virtual Points::iterator end() {
        return points.end();
    }

private:
    Points points;
};

// With adapter we are able to
void drawingLine(vector<Line>::iterator start, vector<Line>::iterator end) {
    for (auto& i = start; i < end; ++i) {
        LineToPointAdapter ltp { i };
        drawing(ltp.begin(), ltp.end());
    }
}
