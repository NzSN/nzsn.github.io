3. FAQ

  • Why is it allowed to pass R-Values by const reference but not by normal reference?

Answer: The C++ language says that a local const reference prolongs the lifetime of temporary values until the end of the containing scope, but saving you the cost of a copy-construction (i.e. if you were to use an local variable instead).

You can find the answer at Page 63 of n4849.

  • What is the template parameter T1 deduced from calling of makepair show following:
template <class T1, class T2>
inline std::pair<T1,T2> make_pair(T1 x, T2 y)
{
    x = 0;
    return std::pair<T1,T2>(x, y);
}

int main() {
    const char (&ref1)[] = "foo";
    const int (&ref2) = 42;

    std::pair<const char*, int> r = make_pair(ref1, ref2);

    return 0;
}

Answer: T1 is deduced to type const char* by calling with argument const char (&). This is defined by standard at page 412 of n4849.
