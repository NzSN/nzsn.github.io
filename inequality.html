2. Inequality

Coq uses the operator “<>” for inequality, which really means “equality is unprovable” or “equality implies False”. 

Notation "x <> y  :> T" := (~ x = y :>T) : type_scope.
Notation "x <> y" := (x <> y :>_) : type_scope.

3. Example

Theorem neq_nega: (forall a, a <> (negb a)).
Proof.
  intros a.
  unfold not.
  case a.
    intros a_eq_neg_a.
    simpl in a_eq_neg_a.
    discriminate a_eq_neg_a.

    intros a_eq_neg_a.
    simpl in a_eq_neg_a.
    discriminate a_eq_neg_a.
Qed.

4. Rule

If you have a hypothesis “<name> : (<constructor1> …) = (<constructor2> …) OR ”<name> : <constant1> = <constant2> Then use the tactic “discriminate <name>.”.

5. Tatic

5.1. discriminate

“discriminate” operates on a hypothesis where values of inductive type are compared using equality. If the constructors used to generate the type are the different, like here where we have “true = false”, then Coq knows that situation can never happen. It’s like a proof of False. In that case, “discriminate <hypname>.”
