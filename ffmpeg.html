1. What is video ?

A video is a collection a picture running at a given rate.

2. What is codec ? Why ?

CODEC is an electronic circuit or software that compresses or decompresses digital audio/video.

Without codec, an video may be too big to streaming.

3. Digital Container

A place to contains video stream and audio streams and metadatas (synchronization and general metadata, e,g, title).

4. What is FFmpeg ?

A cross-platform solution to record, convert and stream audio and video.

4.1. Common video operations

4.1.1. Transcoding

The act of converting one of the stream (audio or video) from one CODEC to another one.

4.1.2. Transmuxing

The act of converting from one format (container) to another one.

4.1.3. Transrating

The act of changing the bit rate, or producing other renditions.

A video may be play on in a 2G connection using a less powerful smartphone or in a fiber internet connection on their 4K TVs.

4.1.4. Transsizing

The act of converting from one resolution to another one.

Reasons are about the same as for the transrating.

4.1.5. Adaptive Streaming

The act of producing many resolutions (bit rates) and split the media into chunks and serve them via http.

This act provide a flexible media that can be watched on a low end smartphone or on a 4K TV, it’s also easy to scale and deploy but it can add latency.

5. Learn FFmpeg libav the Hard Way

5.1. Syncing audio and vidio

5.1.1. Timing

  • Presentation Timestamp (PTS): A value to indicate that when to display.

  • Timebase: Time length of smallest unit of time during performing a stream.

    For example: Meaning of timebase 1/75 is interval of time unit is 1/75 second.

  • Timescale: timebase / 1 is timescale, timescale should be divisible by the frame rate (fps).

5.1.2. PTS and DTS

PTS: timescale / FPS

DTS: Decoding order.

6. How to API

6.1. Open a Media

To open a media you need:

  • Unpack It’s digital container
  • Find out all streams
AVFormatContext *input_format_context = NULL;

// Open media file
avformat_open_input(&input_format_context, "filename", NULL, NULL);

// Sometimes you may want to force parse stream info with specific
// decoder. e.g. force input format with libvpx deocder due to default decoder
// , vp8, unable to recognize alpha channel.
input_format_context->video_codec = avcodec_find_decoder_by_name("libvpx");

// Find Stream infos
// Information of Streams is reside in input_format_context
avformat_find_stream_info(input_format_context, NULL);

6.2. Prepare CoDec

CoDec related structures are:

  • AVCodec
  • AVCodecContext

Each stream of a digital cotnainer have a related CoDec.

6.2.1. Video Encoder

The majority part of prepare an encoder is to

  • Open an Encoder (AVCodec)
  • Setup an Encoder Context (AVCodecContext)

    • Height and Width 

          AVCodecContext *cc;
    cc->height = height;
    cc->width = width;

    • bitrate 

          AVCodecContext *cc;
    cc->bit_rate = bitrate;
    cc->rc_max_rate = max_rate;
    cc->rc_min_rate = min_rate;

    • timebase 

          AVCodecContext *cc;
    cc->time_base = timebase;

    • framerate 

          AVCodecContext *cc;
    cc->framerate = { .num = 1, .den = 30 };

    • Codec ,#+BEGINSRC C++ AVCodec *c; AVCodecContext *cc;

      c = avcodecfindencoderbyname(idofencoder); cc = avcodecalloccontext3(c); if (!c) abort(); avcodecopen2(cc, c); #+ENDSRC

    • Pixel format 

          AVCodecContext *cc;
    cc->pix_fmt = AV_PIX_FMT_ARGB;

6.2.2. Audio Encoder

Setup Audio Encoder require different process. Cause Video Encoder handle things that different from things that deal by Audio Encoder.

Audio datas have their own charaters, e.g, Channels, samplefmt, etc.

Need to setup these value to AVCodecContext.

  • Channels
  • Channellayout
  • Sample rate
  • Sample format
  • Bitrate
  • Timebase
  • etc

6.2.3. Decoder

Just copy stream’s codecpar to AVCodecContext and then open AVCodecContext for AVCodec; 

AVStream *avs;
AVCodec **avc;
AVCodecContext **avcc;

*avc = avcodec_find_decoder(avs->codecpar->codec_id);
if (!*avc) {logging("failed to find the codec"); return -1;}

*avcc = avcodec_alloc_context3(*avc);
if (!*avcc) {logging("failed to alloc memory for codec context"); return -1;}

if (avcodec_parameters_to_context(*avcc, avs->codecpar) < 0) {logging("failed to fill codec context"); return -1;}

if (avcodec_open2(*avcc, *avc, NULL) < 0) {logging("failed to open codec"); return -1;}
