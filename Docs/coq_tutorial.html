2.1. Proof state

You are able to see these proof state by Coqide. 

A : Prop
proof_of_A : A
============================
A

Everything above the bar are called “hypotheses”, as “the context”.

Below the bar is what we are trying to prove. It is called “the current subgoal”.

2.2. Rules

  • RULE: If the subgoal starts with “(forall <name> : <type>, …” Then use tactic “intros <name>.”
  • RULE: If the subgoal starts with “<type> -> …” Then use tactic “intros <name>.”.
  • RULE: If the subgoal matches an hypothesis, Then use tactic “exact <hypname>.”.
  • RULE: If you have subgoal “<goaltype>” AND have hypothesis “<hypname>: <type1> -> <type2> -> … -> <typeN> -> <goaltype>”, Then use tactic “refine (<hypname> _ …).” with N underscores.
  • RULE: If your subgoal is “~<type>” or “~(<term>)” or “(not <term>)”, Then use tactic “intros”.
  • RULE: If the current subgoal contains a function call with all its arguments, Then use the tactic “simpl.”.

3. Proposition

Prop is an important concept, so let’s see some propositions:

  • (forall x : nat, (x < 5) -> (x < 6))
  • (forall x y : nat, x + y = y + x)
  • (forall A : Prop, A -> A)

A prop can be proven or unproven. Cause some true porp can never proven so coq don’t treat a prop as true orfalse.

4. More complicated example

Theorem forward_small : (forall A B : Prop, A -> (A->B) -> B).
Proof.
 intros A.
 intros B.
 intros proof_of_A.
 intros A_implies_B.
 pose (proof_of_B := A_implies_B proof_of_A).
 exact proof_of_B.
Qed.

The command “pose” assigns the result of “AimpliesB proofofA” to the new hypothesis “proofofB”.

RULE: If you have an hypothesis “<hypname>: <type1> -> <type2> -> … -> <resulttype>” OR an hypothesis “<hypname>: (forall <obj1>:<type1>, (forall <obj2>:<type2>, … <resulttype> …))” OR any combination of “->” and “forall”, AND you have hypotheses of type “type1”, “type2”…, Then use tactic “pose” to create something of type “resulttype”.

4.1. Backward proof

a “backward proof” that breaks the goal into smaller and simpler subgoals. 

Theorem forward_small : (forall A B : Prop, A -> (A->B) -> B).
Proof.
  intros A B.
  intros proof_of_A A_implies_B.
  refine (A_implies_B _).
    exact proof_of_A

Theorem backward_huge : (forall A B C : Prop, A -> (A->B) -> (A->B->C) -> C).
Proof.
 intros A B C.
 intros proof_of_A A_implies_B A_imp_B_imp_C.
 refine (A_imp_B_imp_C _ _).
   exact proof_of_A.

   refine (A_implies_B _).
     exact proof_of_A.
Qed.

4.2. Proof going forward (huge)

Theorem forward_huge : (forall A B C : Prop, A -> (A->B) -> (A->B->C) -> C).
Proof.
 intros A B C.
 intros proof_of_A A_implies_B A_imp_B_imp_C.
 pose (proof_of_B := A_implies_B proof_of_A).
 pose (proof_of_C := A_imp_B_imp_C proof_of_A proof_of_B).
 exact proof_of_C.
Show Proof.
Qed.

5. true and false vs. True and False

5.1. Inductive

Inducttive lets you create a new type. 

   Inductive False : Prop := .

   Inductive True : Prop :=
     | I : True.

   Inductive bool : Set :=
     | true : bool
     | false : bool.:w
  • Capital-F “False” is a Prop with no proofs.
  • Capital-T “True” is a Prop that has a single proof called “I”. (That’s a capital-I, not a number 1.)
  • Lastly, “bool” is a Set and “bool” has two elements: lower-case-t “true” and lower-case-f “false”.

5.2. Reduce to ad absurdium

Theorem absurd2 : forall A C : Prop, A -> ~ A -> C.
Proof.
  intros A C.
  intros proof_of_A proof_that_A_cannot_be_proven.
  unfold not in proof_that_A_cannot_be_proven.
  pose (proof_of_False := proof_that_A_cannot_be_proven proof_of_A).
  case proof_of_False.
Qed.
