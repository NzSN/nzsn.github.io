1. Overview

Haskell is a nonstrict language, not a purely lazy language, because you are able to force an expression to be evaluted at some time. Understand how haskell to lazy expressions could help you to write haskell codes.

2. Evaluation Strategies

NonStrict is a evaluation strategy. There are three different evalution strategies:

  • Call by value: It’s a strict way to evaluate expression, an expression that depended from another expression is evaluated if such expression need to evaluated. It’s inside-out evaluatoin.

  • Call by name: Evaluation only happens as needed. An expression able to create a thunk without evaluted and If such expression is evaluated in the future then the result will replace the thunk.

    Nonstrictness include this evaluation strategy. This is outside-in.

  • Call by need: Same as call by name, but expression are only evaluated once. Haskell also include this strategy.

3. Thunk

A Thunk is a computation may be computed in the future. Essentially thunks are computations not yet evaluated up to weak head normal form.

3.1. What not to be thunked ?

In short, expresions that not a constant in Core will be thunked otherwise will not be thunked.

  • Data constructor that fully applied
  • Constant: a concrete value. Something are not a constant but you may think it is, for example:
let a = 2

This expression is not a constant it’s a function cause it’s type is Num a => a. Cause it require one more argument to make sure what it’s the really type of a so It’s a computation and It will be thunked.

3.2. When to stop evaluating

GHC will stop opportunistically evaluating as soon as it hits a computation: 

Prelude> let xs = [1, 2, id 1] :: [Integer]
Prelude> :sprint xs
myList = [1,2,_]

’id 1’ is a function not a data constructor so it will be thunked.

4. Sharing

GHC share results of computations via names. For example: 

let x = 1 :: Int
in x + x

x is evaluated only one time.

4.1. What prevents sharing

  • Inlining expressions: Inlining expressions creates independent thunks that will get computed separately. 

      f :: a -> Int
  f _ = trace "f" 1

  f 'a' -- output: f
        --         1

  f 'a' -- output: f
        --         1
  • named arguments: named arguments make expression as a function not a constant that bind with a name.

-- Named arguments
f :: a -> Int
f a = trace "f" const 1 a

f 'a' -- output: f
                 1

f 'a' -- output: f
                 1
-- No named arguments
f :: a -> Int
f = trace "f" const 1

f 'a' -- output: f
                 1

f 'a' -- output: 1
  • Typeclass constraints: Typeclass expect more arguments to let it to be a concrete type, before that it’s a function. Hence it’s not a name and not be shared.
