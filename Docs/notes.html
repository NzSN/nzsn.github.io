3. Tactics

3.1. induct

[induct x]: where [x] is a variable in the theorem statement, structure the proof by induction on the structure of [x]. You will get one generated subgoal per constructor in the inductive definition of [x]. (Indeed, it is required that [x]’s type was introduced with [Inductive].)

3.2. simplify

[simplify]: simplify throughout the goal, applying the definitions of recursive functions directly. That is, when a subterm matches one of the [match] cases in a defining [Fixpoint], replace with the body of that case, then repeat.

3.3. lineararithmetic

[lineararithmetic]: a complete decision procedure for linear arithmetic. Relevant formulas are essentially those built up from variables and constant natural numbers and integers using only addition, with equality and inequality comparisons on top. (Multiplication by constants is supported, as a shorthand for repeated addition.)

3.4. Chaining

induct e; simplify; lineararithmetic. Oo, look at that! Chaining tactics with semicolon, as in [t1; t2], asks to run [t1] on the goal, then run [t2] on every generated subgoal. This is an essential ingredient for automation.

3.5. cases

[cases e]: break the proof into one case for each constructor that might have been used to build the value of expression [e]. In the special case where [e] essentially has a Boolean type, we consider whether [e] is true or false.

3.6. match

Let’s get fancier about automation, using [match goal] to pattern-match the goal and decide what to do next! The [|-] syntax separates hypotheses and conclusion in a goal. The [context] syntax is for matching against any subterm of a term. The construct [try] is also useful, for attempting a tactic and rolling back the effect if any error is encountered. *)

3.7. rewrite

[rewrite H]: where [H] is a hypothesis or previously proved theorem, establishing [forall x1 .. xN, e1 = e2], find a subterm of the goal that equals [e1], given the right choices of [xi] values, and replace that subterm with [e2].

3.8. transitivity

[transitivity X]: when proving [Y = Z], switch to proving [Y = X] and [X = Z].

3.9. apply

[apply H]: for [H] a hypothesis or previously proved theorem, establishing some fact that matches the structure of the current conclusion, switch to proving [H]’s own hypotheses. This is backwards reasoning via a known fact.

3.10. symmetry

[symmetry]: when proving [X = Y], switch to proving [Y = X].

3.11. unfold

[unfold X]: replace [X] by its definition.

3.12. equality

[equality]: a complete decision procedure for the theory of equality and uninterpreted functions. That is, the goal must follow from only reflexivity, symmetry, transitivity, and congruence of equality, including that functions really do behave as functions.

3.13. propositional

[propositional]: simplify the goal according to the rules of propositional logic.

3.14. firstorder

[firstorder]: simplify first-order logic structure. Be forewarned: this one is especially likely to run forever!

3.15. assumption

[assumption]: prove a conclusion that matches some hypothesis exactly.
