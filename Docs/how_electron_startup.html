2. Question

2.1. How electron load and run application code ?

Electron run node at BrowserProcess that enable user to use JavaScript to control behaviors of electron by execute those javascripts with node.

Hence, before load and run application code, electron have to initialize NodeJS to enable ability to run JavaScript.

  • ElectronBrowserMainParts::PostEarlyInitialization()

OK, NodeJS is initialized in this function. This function is also load application code with following code. 

node_bindings_->LoadEnvironment(node_env_.get());
  • node::StartExecution(Environment*, StartExecutionCallback);
  • node::StartExecution(env, “internal/main/runmainmodule” * const char* *); this cpp function will compile and run the javascript that with path, //thirdparty/electronnode/lib/internal/main/runmainmodule.js.
// run_main_module.js
require('internal/modules/cjs/loader').Module.runMain(mainEntry);

// internal/modules/cjs/loader.js
Module.runMain = require('internal/modules/run_main').executeUserEntryPoint;

Function executeUserEntryPoint use runEntryPointWithESMLoader to load, electron/js2c/browserinit, which is reside in out/XXX/gen.

  • gen/electron/js2c/browserinit.js

browserinit also use runEntryPointWithESMLoader to loade user’s application main module where the entry of application is declared and defined. 

// Finally load app's main.js and transfer control to C++.
if ((packageJson.type === 'module' && !mainStartupScript.endsWith('.cjs')) || mainStartupScript.endsWith('.mjs')) {
    const { runEntryPointWithESMLoader } = require('internal/modules/run_main');
    const main = (__webpack_require__(/*! url */ "url").pathToFileURL)(path.join(packagePath, mainStartupScript));

    // Here main is the path to the main module of user's application.
    runEntryPointWithESMLoader(async (cascadedLoader) => {
        try {
            await cascadedLoader.import(main.toString(), undefined, Object.create(null));
            appCodeLoaded();
        }
        catch (err) {
            appCodeLoaded();
            process.emit('uncaughtException', err);
        }
    });
}
else {
    // Call appCodeLoaded before just for safety, it doesn't matter here as _load is synchronous
    appCodeLoaded();
    Module._load(path.join(packagePath, mainStartupScript), Module, true);
}

2.2. How require work ?

Entry to load user require module is defaultLoadimpl at function loadSource which is defined at //thirdparty/electronnode/lib/internal/modules/cjs/loader.js.

If you want to do something on source of require js, you can do something at Module.extensions[’.js’] that defined at the same file.
