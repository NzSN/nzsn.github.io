Purposes: Prevent unauthorized application code connect to our service.

In short, encrypt main module and all another modules that be required by require() function.

By [0] and [1], we should do descryption of source of encrypted module before the module is translated.

The problem of how to load encrypted module is clear. Next, is how to store the key and strategy to decrypt encrypted module.

1. Design

Declare a format for encrypted module. And all files with the format should be able to be recognized by NodeJS and then NodeJS know it should decrypts files with the format.

Assume that the format is called as ’encrypted-json’, short as ’ejs’.

Decryption is happen at native layer of electron to ensure the performance of decryption process. Where should the native defined ? as v8 builtin? or at electron scope ?

2. Implementation

A new native interface to decrypt JS source. Then the interface used by cjs to load encrypted application code.

The native interface should implement in V8 of nodejs.

3. Use

3.1. Encrypt

openssl aes-256-cbc -e -in main.js -out main.ejs -K "$AES_KEY" -iv "$AES_IV"

where $AESKEY and $AESIV is heax string value.

main.ejs should be able to decrypt by command 

openssl aes-256-cbc -d -in main.ejs -out main.js -K "$AES_KEY" -iv "$AES_IV"

3.2. Decrypt by client

Above command first padding datas with PKCS#7 padding the encrypt. So after decrypt you need to remove padding to get original datas.
