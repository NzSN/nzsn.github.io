Compilation of WASM is begin by the WASM ENGINE that unique for the whole V8 engine, which is defined at //v8/src/wasm/wasm-engine.{h,cc}. There are several ways to compile WASM module:

  1. Sync compilation
  2. Async compilation
  3. Streaming compilation
  4. Lazy compilation

1. Sync Compilation

Entry point for Sync Compilation

v8::internal::wasm::WasmEngine::SyncCompile() at //v8/src/wasm/wasm-engine.cc

1.1. Stages

1.1.1. Decoding

Transform bytes of WASM Module into v8::internal::wasm::WasmModule which is static representat- ation of a WASM Module in V8.

This transformation is done by v8::internal::wasm:ModuleDecoderImpl which implement v8::internal:: wasm::Decoder.

Decoder check the start bytes of bytes of WASM MODULE then decode section by section. Decoding generate a structure that mapping bytes in WASM Module to meaning substructure of the st- ructure that represent WASM MODULE. No copying, just mapping.

The bytes is maintained during compilation.

1.1.2. Compilation

At very first, try find out an existing module that equal to bytes inputed. This is done by hashing for inputed bytes and compare the hash code with codes in module cache.

Otherwise, compilation is triggered.

Steps to compile a wasm module:

  1. Create a new NativeModule, currently, i guess that a NativeModule is a structure that able to run wasm instruction.
  2. (Assume baseline compilation) Then to use Liftoff to compoile wasm instructions into native instructions. Archtecture of system to Transform bytes into NativeModule

  3. Task (BackgroundCompileJob) of compilation is posted to worker thread, current thread is waiting for event that indic- ated the finished of compoilation.

    This process is defined at function v8::internal::wasm:’anonymous’::ExecuteCompilationUnites, this function is called by v8::internal::wasm::BackgroundCompileJob::Run.

    1. Translate instruction by instructions.
    2. Add compiled code to NativeModule. Modified compiled code and copy to new allocated codespace. This process also update relocation information for the compiled code.
    3. Publish compiled code. Let the compiled code accessible from NativeModule
