1. Path to execute a Javascript Function (JSFunction)

Entry to call a JSFunction:

  • v8::internal::Execution::Call
  • v8::internal::Execution::CallScript
  • v8::internal::Execution::CallBuiltin

From these points the following path is draw in some cases:

1.1. BuiltinsJSEntry

Procedure of this builtin as following:

  • Setup frame, to maintain state of machin, for example, push fp and callee-saved (gp,xmm) registers.
  • Maintain precondition of consequence calling of builtins
    • Setup RootRegister, which contain the address to an IsolateRoot its also the address of first byte of Isolate.

    • Backup current values of external reference fields of Isolate (CEntryFP adress and Fast C Caller FP and Fast C Caller PC).

      These fields may related to the current calling state of Javascript function.

    • Setup address of context onto stack
    • Setup stack frame marker to indicate that whether the javascript function correspond to current is the outermost javascript function. If the outer most then push StackFrame::OUTERMOSTJSENTRYFRAME onto stack otherwise StackFrame::INNERJSENTRYFRAME.
  • Then calling the entrytrampoline by call instruction, here the BuiltinsJSEntryTrampoline should be the case.
  • Finally, restore the state of machine and call ret instruction.

1.2. BuiltinsJSEntryTrampoline

The mission of this builtin is to setup an environment that enable consequence builtins to access the called function and arguments of the called function.

All those information is push onto stack frame of BuiltinsJSEntryTrampoline which is shown in layout of Entry Stack.

This builtin will transfer control to target function by calling Builtins::Call() or Builtins::kConstruct.

1.3. BuiltinsCallReceiverIsAny

If BuiltinsJSEntryTrampoline transfer control by Builtins::Call() then this builtin is in the path.

The main job of this builtin is to choose the way to call the target specified by %rdi depend on the instance type field of the map of the target to called.

Instructions of this builtin work like many if statements to select function to call.

1.4. BuiltinsCallFunctionReceiverIsAny

The only thing the function do is convert the target function that reference by %rdi into JSReceiver.

1.4.1. Remark

This procedure seems because of there has no an already prepared JSReceiver to be applied by the function target, hence have to create or convert a new JSReceiver from the function target.

1.5. BuiltinsInterpreterEntryTrampoline

Build up environment for execution of Function:

  1. Current context.
  2. The JSFunction being called.
  3. Reference to BytecodeArray
  4. The offset to the bytecode will be executed soon from the begin of BytecodeArray.
  5. Feedback vector
  6. Register file entries, specifically Interpreter Register not machine register.

In short, this function build up a frame contain informations is required during execution of JSFunction.
