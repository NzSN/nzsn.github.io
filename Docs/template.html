3.2. Template Argument Deduction

Type parameters can be inferenced by the arguments that pass to function or you can explicit declare type of type parameter.

For example: 

template<typename T>
T max(T a, T b) { ... }

max(1, 2);
max<int>(1, 2);
max<int>(1, 2.0); // Passed
max(1, 2.0); // Errored

3.3. Type Conversions During Type Deduction

automatic type conversions are limited during type deduction:

  • Automatic conversions do not apply to type deduction if declareing call parameters by reference.
template<typename T>
void doSomething_(T a, T b) {
    a = b;
}


template<typename T>
void doSomething(T& a, T& b) {
    a = b;
}

int a;
double b;

doSomething_<int>(a, b); // OK
doSomething<int>(a, b);  // Failed
return 0;

  • When declaring call parameters by value, only trivial conversions that decay are supported. 

    template<typename T>
T max (T a, T b);

int const c = 42;
max(i, c);       // OK: T is deduced as int
max(c, c);       // OK: T is deduced as int
int& ir = i;
max(i, ir);      // OK: T is deduced as int
int arr[4];
foo(&i, arr);    // OK: T is deduced as int*

However, the following are errors:

max(4, 7.2);     // ERROR: T can be deduced as int or double
std::string s;
foo("hello", s); // ERROR: T can be deduced as char const[6] or std::string

3.4. Specialization

3.4.1. Full Specification

Specify Template Arguments to a concrete type.

3.4.2. Partial Specification

Specify Templates Arguments to a family of types. 

// partial specialization of class Stack<> for pointers:
template<typename T> class Stack<T*> {
private: std::vector<T*> elems; // elements
public: void push(T*); // push element
    T* pop(); // pop element
    T* top() const; // return top element
    bool empty() const { // return whether the stack is empty
        return elems.empty();
    }
};


// partial specialization: both template parameters have same type
template<typename T> class MyClass<T,T> { ... };
// partial specialization: second type is int
template<typename T> class MyClass<T,int> { ... };
// partial specialization: both template parameters are pointer types
template<typename T1, typename T2> class MyClass<T1*,T2*> { ... };

3.5. Keyword typename

typename is used to declare template argument for a template. And it can also used to declare types that be qualifed by template argument. 

template<typename T> class MyClass {
public:
    ...
void foo() {
        typename T::SubType* ptr;
    }
};

The second typename declare ptr is a pointer point to T::SubType.

6. By Value or by Reference ?

6.1. Passing by Value

6.1.1. Cost

In intuition, Passing by Value is expensive but this is not the case. In fact, Passing by Value is expensive only if the expression you passed to template function is lvalue.

Cost of Copy is optimized away by:

  • Move Semantics
  • Compiler optimizes (Not true before C++17)

But Passing by Reference is required since call with a lvalue expression is a pretty common case.

6.1.2. Decays

Argument you passed to Template Function is decayed before materialization.

6.2. Passing by Reference

Cases of all materialization of Template Functions with Passing by Reference are no copys. Also, passing the argument never decays which make sure Template Functions unable to do something to argument that not allow by the caller.

6.2.1. Passing by Constant Reference

template<typename T>
void printR(T const& arg) { ... }

// Example
std::string returnString();
std::string s = "hi";

printR(s); // 1
print(std::string("hi")); // 2
printR(returnString()); // 3
printR(std::move(s)); // 4

Constant Reference in case 2 and 3 is used to prolong the lifetime of Temporary Object, those objects is destroyed when printR return.

6.2.2. Passing by Nonconstant Refernce

template<typename T>
void outR(T& arg) { ... }

calling outR with prvalue and xvalue usually is not allow. The case that allow to passing rvalue to outR: 

std::string const c = "hi";
outR(c);
outR(returnConstString());

In such cases, arg of outR becoming a declaration of a constant reference. This can be forbided by:

  • staticassert
  • std::enableif
  • concept

detail at Page 111 of C++ templates sec.

6.2.3. Passing by Forwarding Reference

  1. Forwarding references

    Special kinds of references that preserve the value category of a function argument.

    A reference is Forwarding reference if:

    • Function parameter of a Function Template declared as rvalue reference to cv-unqualified type template parameter of that function. 

      template<class T>
int f(T&& x)                      // x is a forwarding reference
{
    return g(std::forward<T>(x)); // and so can be forwarded
}

int main()
{
    int i;
    f(i); // argument is lvalue, calls f<int&>(int&), std::forward<int&>(x) is lvalue
    f(0); // argument is rvalue, calls f<int>(int&&), std::forward<int>(x) is rvalue
}

template<class T>
int g(const T&& x); // x is not a forwarding reference: const T is not cv-unqualified

template<class T>
struct A
{
    template<class U>
    A(T&& x, U&& y, int* p); // x is not a forwarding reference: T is not a
                             // type template parameter of the constructor,
                             // but y is a forwarding reference
};

    • auto&& except when deduced from a brace-enclosed initializer list: 

      auto&& vec = foo();       // foo() may be lvalue or rvalue, vec is a forwarding reference
auto i = std::begin(vec); // works either way
(*i)++;                   // works either way

g(std::forward<decltype(vec)>(vec)); // forwards, preserving value category

for (auto&& x: f())
{
    // x is a forwarding reference; this is the safest way to use range for loops
}

auto&& z = {1, 2, 3}; // *not* a forwarding reference (special case for initializer lists)
  2. How determine reference type of forward ?

    If argument of Function Template is lvalue the lvalue reference otherwise rvalue reference. 

    template<typename T>
void passR(T&& arg) { ... }

std::string s = "hi";
passR(s); // OK: T deduced as std::string& (also the type of arg)
passR(std::string("hi")); // OK: T deduced as std::string, arg is std::string&&
passR(returnString()); // OK: T deduced as std::string, arg is std::string&&
passR(std::move(s)); // OK: T deduced as std::string, arg is std::string&&
passR(arr); // OK: T deduced as int(&)[4] (also the type of arg)

    some suprises: 

    std::string const c ="hi";
passR(c); // OK: T deduced as std::string const&
passR("hi"); // OK: T deduced as char const(&)[3] (also the type of arg)
int arr[4];
passR("hi"); // OK: T deduced as int (&)[4] (also the type of arg)
  3. Drawback
    template<typename T>
void passR(T&& arg) { // arg is a forwarding reference
    T x; // for passed lvalues, x is a reference, which requires an initializer
    ...
}

foo(42); // OK: T deduced as int

int i;
foo(i); // ERROR: T deduced as int&, which makes the declaration of x in passR() invalid

8. Dealing with Return Values

Return a reference from a Template Function is potentially a source of trouble, cause the reference returned may refer to identity that no longer exists.

Cases that need to returning references:

  • Returning elements of container or strings
  • Granting read/write access to class members
  • Returning objects for chained calls

All these cases may cause trouble if used improperly. For example: 

std::string* s = newstd::string( "whatever");
auto& c = (*s)[0];
delete s;
std::cout << c; // run-time ERROR

8.1. Ways to ensure that function templates return by values

For the problems that introduced above, we may need to ensure that Template Function return by values. There are two ways to do that

  • Type trait std::removereference<> to convert type T to a nonreference:
template<typename T>
typename std::remove_reference<T>::type retV(T p) {
    return T{...}; // always returns by value
}
  • auto, because auto always decays:
template<typename T> auto retV(T p) // by-value return type deduced by compiler
{
    return T{...}; // always returns by value
}

9. Template type deduced from function parameter type

#include <iostream>
#include <functional>

template <typename T>
struct parameter_type;

template <typename ReturnType, typename ParameterType>
struct parameter_type<std::function<ReturnType(ParameterType)>> {
    using ParameterT = ParameterType;
};

template <typename T>
using param_t = typename parameter_type<decltype(std::function{std::declval<T>()})>::ParameterT;

template<class T>
void whenMatchesType(T) {
    using Parameter = param_t<T>;
    static_assert(std::is_same_v<Parameter, int>, "Only callables that take int as parameter is allowed");
}

int main() {
    whenMatchesType([](int){});
    //whenMatchesType([](double){});
}
