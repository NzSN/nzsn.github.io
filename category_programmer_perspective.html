1. Notations

  • \(Hom_{C}(a,b)\): A set of morphisms from object a to object b in a category \(C\) is called a hom-set and is written as \(C(a,b)\) (or, sometimes, \(Hom_{A}(a,b)\) )

2. What is Category ?

Category is all about objects and arrows (morphisms), morphisms here not require to be a ’structure-preserving mapping’ it can be just a map or another things.

A Category has two basics properties:

  • Existence of an identity arrow for each object.

  • Arrows should be composable, for example, if \(f : A \rightarrow B\) and \(g : B \rightarrow C\) then there must also exists \(z: A \rightarrow C\).

    Properties of Composition

    1. Asociative: \((h \circ g) \circ f == h \circ (g \circ f)\)
    2. Unit of composition: Suppose \(f: A \rightarrow B\) then \(f \circ id_{A} = f\) and \(id_{B} \circ f = f\) .

A set of morphisms from object \(a\) to object \(b\) in category \(\mathbf{C}\) is called a hom-set and is written as \(\mathbf{C}(a,b)\).

2.1.2. Order

A collection \(G\) of objects and \(\le\) relation is indeed a category. Let’s check that:

  • Existence of Identity morphism, \(\forall a \in G: a \le a\), true !
  • Composability of morphism, \(\forall a,b,c \in G: a \le b \land b \le c \Rightarrow a \le c\)
  • Associcativity of composition which is certainly true

Hence, \(G = (G,\le)\) is a category.

2.2. Duality

If there exist a category \(\mathbf{C}\) then we are free to get another category \(\mathbf{C}_{R}\) by reverse all arrow within the category.

2.2.1. Asymmetry

Property of different between Category A and Category B, B is built from A by reverse morphisms in A.

2.3. Isomorphisms

Two object (A and B) are isomorphic if :

  • \(f \circ g = id\) (1)
  • \(g \circ f = id\) (2)

This implies f is a bijective. In Category Theory for Programmers an isomorphism is an invertible morphism.

It’s only need to satisfies one-to-one mapping property, structure preserving is not required.

If structure-preserving is required then following property is need:

  • \(f(a \odot b) = f(a) \odot f(b)\)

which implies that \(f\) is a homomorphism for \(\mathbf{A}\) and \(\mathbf{B}\). And also (1) and (2) is injective so \(f\) is also isomorphisms hence \(\mathbf{A}\) and \(\mathbf{B}\) are homomorphic.

3. Monoid as Set (Perspective from set theory)

Ok, first to see the definition of Monoid in Math:

A set \(S := (S, \odot)\) is a monoid if it satisfies the following two axioms:

  1. Associativity \(\forall a,b,c \in S: (a \odot b) \odot c = a \odot (b \odot c)\)
  2. Identity element \(\exists e \in S, \forall a \in S: a \odot e = e \odot a = a\)

In other wors, a monoid is a semigroup with \(e\).

Back to area of programming, List is a monoid. Let \(L := (L, :)\) be set of all lists with a operation that concate two lists. Next we prove that it’s a monoid.

Let \((:) : L \times L \rightarrow L\) be an operator such that \(\forall a \in L, b \in L: a \subseteq (a:b) \land b \subseteq (a:b)\).

  1. Checking of associativity:

    Let \(L_1,L_2,L_3\) be lists and have

    \(L_l = (L_1 : L_2) : L_3\) \(L_r = L_1 : (L_2 : L_3)\)

    \(L_l = L_r\) is got directly from the definition of \(:\) . 2. [] is the identity element of \(L\) .

Hence, \(L\) is a monoid .

5. Products and Coproducts

In this chapter, the main goal is to apply universal construction to define objects for patterns.

Universal construction is a way defining objects in terms of their relationships. Pick a pattern and look for all its occurrences in the category.

5.1. Initial Object

First example to apply universal construction to define object.

The Object that has one and only one morphism going to any object in the category, this object is unique up to isomorphism.

Example of initial object (in Category of Types) : 

absurd :: Void -> a
def absurd[A]: Nothing => A

5.2. Terminal Object

The terminal object is the object with one and only one morphism coming to it from any object in the category.

Example (in Category of Types): 

unit :: a -> ()
unit _ = ()

def unit[A]: A => Unit = _ => ()

Also terminal object is unique up to isomorphism.

5.3. Products

This section apply universal construction to product. Our goal is to generalize product to categories.

Product is a pair of elements with two functins, fst and snd. 

fst :: (a,b) -> a
snd :: (a,b) -> b

Next, define a pattern like this in category of sets. 

p :: c -> a
q :: c -> b

5.3.1. Examples

Let p and q are morphisms of Category of Types. In this example, Int can be considered a candidate for the product of Int and Bool. 

p :: Int -> Int
p x = x

q :: Int -> Bool
q _ = True

p :: (Int, Int, Bool) -> Int
p (x, _, _) = x

q :: (Int, Int, Bool) -> Bool
q (_, _, b) = b

Now let’s rank these two instances of product pattern.

Ranking criterion:

c is “better” than c’ if there is a morphism m from c’ to c.

cause p’ and q’ can be represent by p and q.

Finally, product on categories:

A product of two objects a and b is the object c equipped with two projections such that for any other object c’ equaipped with two projections there is a unique morphism m from c’ to c that factorizes those projections.

5.4. Coproduct

Dual of product. 

i :: a -> c
j :: b -> c

The ranking is also inverted: object c is “better” than object c’ that is equipped with the injections i’ and j’ if there is a morphism m from c to c’ that factorizes the injections

Definition of coproduct:

A coproduct of two objects a and b is the object c equipped with two injections such that for any other object c’ equaipped with two injections there is a unique morphism m from c to c’ that factorizes those injections.

5.6. Functoriality

5.6.1. What is functorial ?

Basically, when you have some expression \(E\) that contains \(F\), and we say \(E\) is functorial in \(F\), it means that \(F \rightarrow E\) , is a functor.

5.6.2. BiFunctors

\(bifunctor :: C \times D \rightarrow E\) where \(C,D,E \in \mathbf{Cat}\)

Bifunctor also mapping morphism of category \(C \times D\) to \(E\). Morphism of \(C \times D\) if just a pair of morphism from category \(C \times D\). These pairs of morphisms can be composed in the obvious way:

\((f,g) \circ (f',g') = (f \circ f', g \circ g')\)

The composition is associative and it has an identity, (id, id). So a Cartesian product of categories is indeed a category.

5.6.3. Product and Coproduct Bifunctors

In previous section we have already defined categorical product. If the product exists for any pair of objects, the mapping from those objects to the product is bifunctorial.

Here’s the Bifunctor instance for a pair constructor, the simplest product type: 

instance Bifunctor (,) where
  bimap :: (a -> c) -> (b -> d) -> (a, b) -> (c, d)
  bimap f g (x, y) = (f x, g y)

By duality, a coproduct exists, if it’s defined for every pair of objects in a category, is also a bifunctor. 

instance Bifunctor Either where
  bimap f _ (Left x) = Left (f x)
  bimap _ g (Right y) = Right (g y)

If treat bifunctorial as functorial, then product and coproduct are both functorial.

5.6.4. Functorial Algebraic Data Types

data Maybe a = Nothing | Just a
-- Equivalent form
type Maybe a = Either (Const () a) (Identity a)

In the second case, Maybe is defined from Bifunctor Either and two functor, Const () and Identity.

5.7. Covariant and Contravariant Functors

Covariant functor is regular functor. Contravariant Functor are functors that maps objects the same way regular functor does, but when it comes to mapping morphisms, it reverses them. 

class Contravariant f where
  contramap :: (b -> a) -> (f a -> f b)

5.8. Profunctors

Function-arrow operator that first argument is contravariant and covariant in the second is called a profunctor.

\(\mathbf{C}^{op} \times \mathbf{D} \rightarrow \mathbf{Set}\)

5.9. Hom-Functor

Description on wiki is more easily to understand.

Suppose C is Hom-Functor then we have: 

-- Mapping on morphisms
f :: X -> Y
g :: X -> Y

C(f,g) :: C(Y,X) -> C(X,Y)

For all objects A and B in C we define two functors to the category of sets as follows (show only mapping on objects here, mapping on morphisms is shown above) :

  • Covariant functor Hom(A, _) maps each object X in C to the set of morphisms, Hom(A, X)
  • Contravariant functor Hom(_, A) maps each object X in C to the set of morphisms, Hom(X, A)
