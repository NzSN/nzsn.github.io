1. Introduction

In short, PatternMatching is a technique that to find out subtrees that specified by a pattern.

When doing program transformation, need to match a parsetree by PatternMatching to detemine where to transform. There are many other cases to apply PatternMatching.

2. Problem Definitions

Formally definition of The Tree-Matching Problem.

\begin{Def}[$\Sigma\text{-term}$] Suppose $\Sigma$ is a finite ranked alphabet $\Sigma$ of function symbols, including constans as nullary functions. $S$ denotes the set of $\Sigma\text{-term}$, formally defined as follows. 1. $\forall b \in \Sigma$ of rank 0, $b$ is a $\Sigma\text{-term}$. 2. If $a$ is a symbol of rank $q$ in $\Sigma$, then $a(t_{1},...,t_{q})$ is a $\Sigma\text{-term}$ provided each of the $t_{i}$ is . 3. Nothing else is a $\Sigma\text{-term}$. \end{Def}

\newpage For example, \(a(a(b,b),b)\) is a tree of following figure. 

     a
    / \
   a   b
  / \
 b   b

\(v\) is a special nullary symbol that serve as placeholder for any \(\Sigma\text{-term}\). \(S_{v}\) denotes the set of \(\Sigma \cup \{v\}\text{-terms}\) .

\begin{Def}[Pattern] A tree pattern is any term in $S_{v}$. if $b(t_{1},...,t_{q})$ is a term, then define $son_{L}(b(t_{1},...,t_{q}))$ to be $t_{i}$ for $1 \leq i \leq q$. \end{Def}

Next to explain how tree patterns are to be matched in \(\Sigma\text{-terms}\):

\begin{Def}[Matching] A pattern p in $S_{v}$ with k occurrences of the symbol $v$ matches a subject tree $t$ in $S$ at node $n$ if there exist $\Sigma\text{-trees}$ $t_{1},...,t_{k}$ in $S$ such that the $\Sigma\text{-term}$ $p^{'}$, obtained from $p$ by substituting $t_i$ for the $i\text{-th}$ occurrence of $v$ in $p$, is equal to the subtree of $t$ rooted at $n$. \end{Def} \begin{Def}[The Matching Problem] A matching problem consists of a finite set of patterns \\ $p_1,...,p_k \in S_v$ and a subject tree t in $S$. A solution to a matching problem is a list of all the \\ pairs $(n, p_i)$ where n is a node in t and $p_i$ matchds at node n. \end{Def} \begin{Def}[Size and Height] The size of a tree is the total number of subtrees (equivalently, nodes) in it. \\ The size of a forest is the sum of all trees in it. The height of a tree is the \\ number of edges in a longest path from the root to a leaf of the tree. \end{Def}
