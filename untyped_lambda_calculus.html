1. Overview

Lambda-Calculus is about behaviour of functions in the simplest, most abstract view.

Only fews of concept remain in lambda-calculus:

  • Variables
  • Construction Principles: Abstraction and application
  • Evaluation Rule: Beta-Reduction

2. Categories of \(\lambda\text{-calculus}\)

  • Untyped lambda calculus
  • Simply typed lambda calculus
  • Second order typed lambda calculus
  • Types denpendent on types (Generalised Types)

Untyped lambda calculus is the most simple one which provide definitions about basic outline of how a lambda look like.

There are three major parts of content defined by Untyped lambda calculus:

  • Structure
  • Relation
  • Evaluation

with these you are able to construct an untyped lambda and apply it to some term.

Simply typed lambda calculus is built on Untyped lambda calculus. Former supply extra structures, Context, Judgement. With those extra structures we are able to do Derivation (Formal proof). More structures more definitions, those extra structures require exra definitions, theorems, lemma.

Simply typed lambda calculus’s input is restricted. You only be able to pass a term to a lambda which type of argument match with input term.

Simply typed lambda calculus is too restricted. We are unable to present some function by lambda of this category. For example, A function that accept any type of terms as its argument.

For the problem show above Second order typed lambda calculus is required. Which provide terms depending on types (second order abstraction), and Simply typed lambda calculus’s provide abstraction terms depending on terms (First order abstraction).

Second order typed lambda calculus provide generalised terms by provide an extra type argument. In a similar manner, able to construct generalised types by provide an extra type argument to lambda calculus which value is a type.

For example,

  • \((\lambda \alpha: *. \alpha \rightarrow \alpha)\beta \rightarrow_{\beta} (\beta \rightarrow \beta)\),
  • \((\lambda \alpha: *. \alpha \rightarrow \alpha)\beta \rightarrow_{\beta} (\gamma \rightarrow \gamma)\),

As you seen, both of those lambda construct types, \(\beta \rightarrow \beta\) and \(\gamma \rightarrow \gamma\). With those expression we are able to construct types in above pattern. Another name of those expressions are type constructor.

As we restrict normal lambda calculus to get valid expressions, We also want to give types to those type cosntructor to ensure final result of type constructor is what we want.

People offer a new word to the type of type constructor, kinds. The most simply kind is \(*\) which meaning is ’any types’. For example,

  • \(\alpha : *\)
  • \(\alpha \rightarrow \alpha : *\)

Suppose there exist a type constructor, \(\lambda \alpha: *. \alpha \rightarrow \alpha\), and It create a arrow type, \(* \rightarrow *\), from a type variable, \(*\), so It’s type is \(* \rightarrow (* \rightarrow *)\). The complete type constructor can be write as follow:

  • \(\lambda \alpha: *. \alpha \rightarrow \alpha: * \rightarrow (* \rightarrow *)\)
