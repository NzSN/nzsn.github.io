2. Terminology

  • DA-Layer: Environment that only Instructions can be evaluated.
  • Python-Layer: Environment that only python can be evaluated.
  • DA-Scope: Scopes in Python-Layer that unable to evaluated by python interpreter.
  • Python-Scope: Scopes in Python-Layer that able to evaluated by python interpreter.
  • Python-AST: Python abastract syntax tree
  • DA-AST: DAL abastract syntax tree

3. Able ?

DAL’s primitives is Instructions. There are several Instructions

  • Condition Instructions (CI)
  • Operations Instructions (OI)
  • Variable Instructions (VI)

CI provide selection structure and loop structures to DAL, OI provide ability to user to operate external entities. VI provide memory store and access to DAL.

In another words, DAL is a language which main purposes is to control external entities by some logics, these logics contain selection structure, loop structures.

Let’s look at Python Code. Obviously python code able to simulate DAL so there should be a way to transform Pythong code into DAL.
