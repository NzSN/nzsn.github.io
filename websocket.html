2.4. Close the Connection

Close the Connection means an endpoint closes the underlying TCP connection, the connection should closed cleanly.

Stages for WebSocket Closing Handshake:

  • Start the WebSocket Closing Handshake (Send a Close control frame)
  • WebSocket Closing Handshake is Started (Either sending or receiving a Close control frame)
  • The WebSocket Connection is Closed (TCP connection is closed): if TCP connection is closed after WebSocket connection closed, the WebSocket connection close is cleanly.

5. Extension

WebSocket specification provide ways to add new extensions.And now an extension that I have to implement is Compression Extensions for WebSocket. Ways to help attach extensions can be found in RFC6455.

Which Extension to used is determined by negotiation between Client and Server. Such negotiation carry on as following ways:

  • Client including a |Sec-WebSocket-Extensions| header filed in Handshake.
  • If Server accepts one or more extensions then such server will including correspond extensions in |Sec-WebSocket-Extensions| header field.

Note that the order of extensions is significant. The first options listed being most preferable.

For example, if there are two extensions “foo” and “bar” and if the header field |Sec-WebSocket-Extensions| sent by the server has the value “foo, bar”, then operations on the data will be made as bar(foo(data)), be those changes to the data itself (such as compression) or changes to the framing that may “stack”.
