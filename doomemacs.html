1. Install local packages

That’s an unconventional use for package!, and will break in subtle ways that are a tad long to list here. I don’t think this is what you want to do, so here are a few alternatives:

1.1. Using load!

  1. Put the package in ~/.doom.d/lisp/my-private-package.el
  2. And from ~/.doom.d/config.el load it with (load! “lisp/my-private-package”)

1.2. Using require

Assuming it’s a package with proper structure (has all the required package headers and a (provide ’my-private-package)), then:

  1. Put the package in ~/.doom.d/lisp/my-private-package.el
  2. In ~/.doom.d/config.el, add ~/.doom.d/lisp to load-path: (add-load-path “lisp”)
  3. And require it (require ’my-private-package).

1.3. Use use-package! (or use-package, either is fine)

The same as no. 2, except you load it with use-package! instead of require: 

(use-package! my-private-package
  :config
  ;; configure your package here
  )

You can also use the :load-path property: 

(use-package! my-private-package
  :load-path "path/to/my/local/package"
  :config
  ;; configure your package here
  )

1.4. Symlink/copy packages into subfolders

Copy or symlink packages into subdirectories under ~/.doom.d/lisp, e.g. 

~/.doom.d/
    packages/
        package-1/*.el
        package-2/*.el
        package-3/*.el

Add these packages to your load-path: 

(let ((default-directory (expand-file-name "packages" doom-private-dir)))
  (normal-top-level-add-subdirs-to-load-path))

And now they can be loaded with require or def-package!/use-package.
