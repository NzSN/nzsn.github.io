2. Introduction

We need some make some abstraction from objects that the system operate with, if not the system may unable to done it’s tasks.

2.1. Machine

Machine is the most important abstraction cause it represent the object that be tested or objects that do test.

Let’s give a formal definition to Machine. Machine is a triple \(\), ID means identifier, prop is Properties, OP is operation that this machine able to do.

For example, representation in python layer is : 

     ###############################################################################
     #                  BoxMachine, a machine that simulate a box                  #
     ###############################################################################
     # Box Properties
     boxProperties = [
         core.Property("contain", 10)
     ]

     boxOpSpec = [
         # Open
         core.OpSpec("open", (), None),
         # Close
         core.OpSpec("close", (), None),
         # Put
         core.OpSpec("put", ("Candy", str), None),
         # Get
         core.OpSpec("get", (), ("Candy", str))
     ]


     class BoxMachine(core.Machine):
         """
         A machine that can open and close, just like a box.
         """

         def __init__(self) -> None:
             core.Machine.__init__(self, "Box", boxProperties, boxOpSpec)

         @core.Machine.Operation
         def open(self) -> core.Operation:
             return core.Operation("core", "Box", core.opTuple("open", None))

         @core.Machine.Operation
         def close(self) -> core.Operation:
             return core.Operation("core", "Box", core.opTuple("close", None))

         @core.Machine.Operation
         def put(self, candy: str) -> core.Operation:
             return core.Operation("core", "Box", core.opTuple("put", candy))

         @core.Machine.Operation
         def get(self) -> core.Operation:
             return core.Operation("core", "Box", core.opTuple("get", None))

Of course, systemt unable to directly perform this python script, need to compile first then performed in DA layer.

A Machine is an abstraction of some aspects of real machines. It has properties and operations. With this abstraction we are able to operate real machines to do some actions to affect real world.

2.2. Property

Property is apart of machine, it represent features of a machine and it also provide details about a feature, such as Whant things that a box machine able hold and how many of things can be contained, with Property we are able to shape an abstraction machine like a real machine.

2.3. Operation specification

Operation specification specify what action that a machine able to perform. Operation may affect the property of a machine or properties may affect some operations, in another words, operation may do read or write to some properties.

An operation is essentially a message in DA layer, it’s a message that from ’core’ to ’real machines’.

User able to define more concrete operation by exten class ’Operation’, such as Config, Query.
