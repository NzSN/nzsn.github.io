1. Introduction

Strategies is about how to apply Rewrite Rules to matched terms in Rewrite System, for example, software for Program Transformation.

2. Fundamental operation for Rewrite Rules

2.1. Sequential composition

\(s_{1}; s_{2}\)

2.2. Non-deterministic Choose

\(s_{1} + s_{2}\)

2.3. Deterministic Choose

\(s_{1} \pm s_{2}\)

2.4. Repeadly

\(\mu(s)\), for example, \(\mu((s_{1}+s_{2}) \pm (s_{3};x))\), repeadly apply \(s_{3}\) until \(s_{1}\) and \(s_{2}\) is applicabale.

2.5. Identity

\(\epsilon\) is always succeeds.

Optional strategy can be builded from Identity, \(s_{1} \pm \epsilon\), try to apply \(s_{1}\) if not applicabale then just succeeds with \(\epsilon\).

2.6. Test

test s, test s without having the transforming effect of s.

3. Redex and Normal Form

If \(t \rightarrow_{s} t'\) then t is s-redex. If \(t \rightarrow_{s} \uparrow\) then t is s-normal form.

4. Backtracking

For those strategies that has more than one candidates rules backtracking is in used. For example, strategy \(s_{1} + s_{2}\) apply one of \(s_{1}\) and \(s_{2}\), try one of them first if fails then backtrack to another one.

5. Term Traversal

When a Rewrite Rule’s right side contains Rwrite Rule then you need to traverse into it to apply all of it’s transform effects.
