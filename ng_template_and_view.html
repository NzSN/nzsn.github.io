Ng-Components

2. Templates and views

Template define view of components. Template is a for of HTML and Angular markup that tells Angular how to render the component.

View hierarchy:

{ <Root comp> <Root template> }

 
<ChildA comp> <ChildA template>   <ChildB comp> < ChildB template>

3. Data binding

Data binding provide ability to transfer data between component and their view.

3.1. Interpolation

COMPONENT –— {{value}} –—> DOM

3.2. Property binding

COMPONENT –— [property] = “value” –—> DOM

3.3. Event binding

COMPONENT <-— (event) = “handler” –— DOM

3.4. Two way binding

COMPONENT <-— [(ngModel)] = “property” –—> DOM

4. Pipes

Angular pipes let you declare display-value transformations in your template HTML. @Pipe decorator defines a function that transforms input values to output values for display in a view. 

<!-- Default format: output 'Jun 15, 2015'-->
 <p>Today is {{today | date}}</p>

<!-- fullDate format: output 'Monday, June 15, 2015'-->
<p>The date is {{today | date:'fullDate'}}</p>

 <!-- shortTime format: output '9:43 AM'-->
 <p>The time is {{today | date:'shortTime'}}</p>

5. Directives

Guid how to render templates. A directive is a class with @Directive() decorator. A component is technically a directive.

In addition to components, there are two other kinds of directives:

  • Structural
  • attribute

And user can define their own directive using the @Directive() decorator.

5.1. Structural

Alter layout by adding, removing, and replacing elements in the DOM.

  • *ngFor is an iterative
  • *ngIf is a conditional

6. Attribute directives

Attribute alter the appearance or behavior of an existing element.

6.1. ngModel directive

The ngModel directive, which implements two-way data binding, is an example of an attribute directive. ngModel modifies the behavior of an existing element (typically <input>) by setting its display value property and responding to change events. 

<input [(ngModel)]="hero.name">

Angular has more pre-defined directives that either alter the layout structure (for example, ngSwitch) or modify aspects of DOM elements and components (for example, ngStyle and ngClass).
