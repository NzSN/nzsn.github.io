1. One Definition Rule

This rule apply to entities in cpp,

  • variable
  • function
  • class type
  • enumeration type
  • concept (since C++20)
  • template

Rule is defined as:

  • Only one definition of enetity is allowed in any one translation unit.
  • Only one definition of non-inlined entity that is odr-used in the entire program. (Compiler not required to check it but it’s undefined behavior).
  • More rules about multi definition in program scope:
    • each definition appears in a different translation unit
    • appears in the same header file
    • name lookup from within each definition finds the same entities (after overload-resolution
    • [linkage] corresponding entities have the same language linkage in each definition
    • [const entity] if a const object is constant-initialized in any of the definitions, it is constant-initialized in each definition
    • Definitions should consistent.

2. ODR-USE

Informally,

  • an object is odr-used if its value is read (unless it is a compile time constant) or written, its address is taken, or a reference is bound to it,
  • a reference is odr-used if it is used and its referent is not known at compile time,
  • a function is odr-used if a function call to it is made or its address is taken.

In short, reference related operation on entity require the entity to be defined. 

struct S
{
    static const int x = 0; // static data member
    // a definition outside of class is required if it is odr-used
    // cause declare static data member inside of class is hardcode
    // and it's in read-only segment.
};

const int& f(const int& r);

int n = b ? (1, S::x) // S::x is not odr-used here
          : f(S::x);  // S::x is odr-used here: a definition is required
