2. Introduction

H.264 Syntax is a rule or a format that need to obey during representing a H.264 video by sequence.

H.264 Syntax is a Hierarchical syntax, from the top level, the sequence level, down through frames and slices to macroblocks and blocks.

3. Terminology

Access Unit: A coded frame or field, is made up of one or more slices.

H.264/AVC Frame: An array of luma samples and two corresponding arrays of chroma samples.

H.264/AVC Field: such as TopField or BottomField.

Picture: Referes collectively to a frame or a field.

4. Syntax Overview

H.264 Syntax is a Hierarchical syntax which means It have some layers as show follow (from high to low layer):

  • Network Abstraction Layer: consists of series of NAL Units.

    Sequence Parameter sets (SPS) and Picture Parameter Sets (PPS) are NAL Units.

    Coded video data is communicated in Video Coding Layer VCL NAL Units, known as coded slices.

  • Slice Layer: Each slice consists of a Slice Header and Slice Data. Slice Data is a series of coded macroblocks (MB).
  • Macroblock Layer: Each macroblock contains the following syntax elements:
    • MB type: I/intra coded or P/inter, B/inter
    • Prediction information
    • Coded Block Pattern (CBP)
    • Quantization Parameter (QP), for macroblocks with CBP != 0.
    • Residual data, for blocks containing non-zero residual coeffcients.

5. Frame, Field and Picture

There are three different orders of pictures:

  • Decoding Order
  • Display Order
  • Reference Order

5.1. Decoding Order

Decoding Order is determined by the parameter framenum, which decoded from the slice header.

framenum for each decoded frame is increase by 1 compared with the previous reference frame in decoding order.

5.2. Display Order

Determined by the POC parameters:

  • ToPFieldOrderCount
  • BottomFieldOrderCount

which are derived from slice header using one of three methods :

5.2.1. Type 0: Slice header with least significant bits of POC.

Display Order is determine by POC LSBs. framenum is increase after a Reference frame.

e.g.

Slice Type Reference? framenum POC LSBs Display Order
1 I Y 0 0 0
2 P Y 1 4 2
3 B N 2 2 1
4 P Y 2 8 4
5 B N 3 6 3
6 P Y 3 12 6
7 B N 4 10 5
8 P Y 4 16 8

5.2.2. Type 1: Cyclic Increament

Interval between two reference frame is a ’cycle’. POCs of two consecutive reference frame form an interval, POC of non-reference frames must within this interval by cyclic.

The ’cycle’ is set up in the sequence parameter set.

e.g.

Slice Type Reference framenum Delta POC Offset POC Display Order
1 I Y 0 0 0 0 0
2 P Y 1 0 +6 6 3
3 B N 2 0 -4 2 1
4 B N 2 2 -4+2 4 2
5 P Y 2 0 +6 12 6
6 B N 3 0 -4 8 4
7 B N 3 2 -4+2 10 5
8 P Y 3 0 +6 18 9

The 4th POC is evaluate in the way like: [(2 - 4) = -2 = 2 <cause of cyclic>] + 2 = 4

5.3. Reference order

Reference order

5.3.1. Reference picture lists

A picture that is coded and available for reference is stored in the Decoded Picture Buffer(DPB) and marked as one of the following:

  • A short term reference picture, indexed according to framenum or POC

  • A Long term reference picture, indexed according to LongTermPicNum, a reference number that derived from parameter LongTermFrameIdx which is assigned when a picture is marked as a long term reference picture.

    A short term reference picture may be changed to long term reference picture by assigned a LongTermFrameIdx at a later time..

Short term reference pictures are able to removed from the DPB by

  • explicit command in the bitstream or
  • when the DPB is full and an automatic mode of DPB handling is in use, in which case the oldest short term picture is removed.

Long term reference pictures can only be removed by explicit command in the bitstream.

  1. Defalut reference picture list order

    Reference Picture list is ordered prior to encoding or decoding. There are two lists, list0 and list1. P-slice use Reference Picture List 0, B-Slice use both list0 and list1.

    Short Term Reference Pictures are listed first and Long Term Reference Pictures follows.

    List Order is important because the earlier in the list require fewer bits to signal. So the default order are organized so that Reference Pictures temporal ’nearer’ to the current picture occur earlier in the list since they are likely to contain the best prediction match for the current picture.

    Order of List0:

    • List0(P-Slice): Decreasing order of PicNum

      PicNum = frame % MaxFrameNum

    • List0(B-Slice): (1) Decreasing order of POC (2) Increasing order of POC

    Order of List1:

    • List1(B-Slice): (1) Increasing order of POC (2) Decreasing order of POC
  2. Frame and Field coding
    1. Picture Adaptive Frame Field Coding (PAFF)
    2. Macroblocks Adaptive Frame Field (MBAFF)

6. NAL Unit (Netwokr Abstraction Layer Units)

There are two types of NAL Units:

  • VCL NAL Units:
    • Coded slice, IDR.
    • Coded slice, non-IDR
    • coded slice data partition
  • Non-VCL NAL Units:
    • Sequence Parameter Sets
    • Picture Parameter Sets

7. Slice layer

7.1. Slice Types

7.1.1. I Slice (Including IDR)

Intra Prediction Only

Contains Only I macroblocks.

7.1.2. P Slice

Intra Prediction (I) and/or prediction from one reference per macroblock partition (P).

Contains I and/or P macroblocks.

7.1.3. B Slice

Intra Prediction (I), prediction from one reference (P) or biprediction, i.e prediction from two reference (B).

Contains I, P and/or B macroblocks.

7.1.4. SP Slice

Switching P slice

Contains P and/or I macroblocks.

7.1.5. SI Slice

Switching I slice

Contains SI macroblocks.

8. Macroblock layer

The Macroblock Layer contains all the syntax elements necessary to decode a single macroblock.
