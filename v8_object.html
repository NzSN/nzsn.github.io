2. Object Model

2.1. Important Structures

2.1.1. Object References

Class Object represent an object reference. It just and wrapper of Address, anything that can point into the heap is an Address not a pointer type. 

|  TaggedImpl  | <---inher--- | Object |
|--------------|
| ptr_: Address|

2.1.2. References to Object References

Represent by ObjectSlot, also a wrapper class around an Address, giving a different meaning to the wrapped Address value ( specifically: it’s kPointerSize-aligned, and you can dereference it to get another Address. Don’t dereference it manually though, use the public interface to get an Object ).

2.1.3. Handles

GC-Safe Object References. Handles are templatized to the type they are wrapping, e.g. Handle<Object> or Handle<Map> .
