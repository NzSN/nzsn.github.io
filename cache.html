1. Processor Cache

1.1. Definition

Fast, small and expensive memory that use to conquer bandwidth gap between processor and memory.

Cache is one layer of Memory Hierarchy.

1.2. Performance measure factors

1.2.1. Miss rate

Categories of Miss rate:

  • Compulsory
  • Capacity
  • Conflict

1.3. Equations

Factors that use to determine memory access performance:

  • Cache access time
  • Miss rate
  • Miss penalty

Expand \(CPU_{time}\) to include memory stall as a factor:

\(CPU_{time} = (\text{CPU clock cycles} + \text{memory stall cycles}) \times \text{Clock cycle time}\)

and Memory stall cycles is represend as

\begin{align} \text{memory stall cycles} = {}& \text{Number of misses} \times \text{Miss penalty}\\ = & IC \times \frac{\text{Misses}}{\text{Instruction}} \times \text{Miss penalty}\\ = & IC \times \frac{\text{Memory accesses}}{\text{Instruction}} \times \text{Miss rate} \times \text{Miss penalty} \end{align}

1.4. How a search a cache line ?

In short

  • Cache line with Address tag to match address from processor
  • Cache line search in parallel.

1.4.1. Processor Address and Cache

Processor Address is divided into:

  • Block Address
    • Tag
    • Index
  • Block Offset

Processor Address is divided into Block Address and Block Offset and Block Address is further divided into Tag filed and Index field.

Tag field is used to match a cache line of a set that indexed by Index field of Block address. Finally, processor use Block offset to acces expected datas of a cache line.

1.5. Strategies about cache miss

Strategies to deal with cache miss:

  • Random: selected candidate blocks randomly.
  • Least recently used (LRU):
  • First in, first out (FIFO)

1.6. Cache write

Write to cache is an heavier operation than read, cause of cache coherency and it may required to change content of cache. Read of cache can begin at the same time with tag checking but write can not, write must waiting until tag matched.

1.6.1. Write Policy

  • Write through
  • Write back
